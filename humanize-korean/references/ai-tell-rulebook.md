# 한국어 AI 문체 통합 룰북

AI 작성 여부를 증명하는 목록이 아니라 **자연스러운 한국어 윤문을 위한 편집 신호**다. 단일 패턴이 아니라 반복·군집·장르 부적합을 근거로 삼는다.

## 심각도

- **강**: 문법적 번역투, 이중 피동, 상투적 결산처럼 한두 번만으로도 어색한 신호.
- **중**: 반복될 때 기계적인 인상을 만드는 신호. 기본 강도에서는 같은 문서에 3회 이상일 때 주로 수정한다.
- **약**: 취향·장르 의존 신호. 적극 강도에서도 다른 신호와 겹칠 때만 수정한다.

## A. 번역투·직역 구조

| ID | 신호 | 게이트 | 안전한 처방 |
| --- | --- | --- | --- |
| A-1 | `~에 대해(서)` 반복 | 주제 의미가 꼭 필요한지 확인 | 목적격 조사나 직접 서술로 단순화 |
| A-2 | `~을 통해/통하여` 반복 | 실제 매개·경로면 보존 | `~로`, `~해서` 등 문맥 표현 |
| A-3 | `~에 있어(서)` | 법률·격식 문맥 여부 | `~에서`, `~을 보면` |
| A-4 | `~와 관련하여`, `~에 기반하여` 반복 | 전문 용례면 보존 | 짧은 조사·동사로 환원 |
| A-5 | `가지고 있다`, `회의를 가지다` | 실제 소유 의미면 보존 | 형용사·동사로 환원 |
| A-6 | `~되어지다`, 불필요한 `~지게 되다` | 피동·과정 의미 확인 | 단일 피동 또는 능동 |
| A-7 | `~에 의해` 피동 반복 | 행위자 은폐가 필요한 문서인지 확인 | 행위자를 주어로 전환 |
| A-8 | `그/그녀/그것/그들` 직역 반복 | 지시 대상이 모호해지지 않는지 확인 | 생략 또는 명사·호칭 복원 |
| A-9 | `~에서의/~에로의/~으로의/~에의` | 굳어진 전문 용례인지 확인 | 절이나 짧은 조사로 풀기 |
| A-10 | 명사 앞 긴 관형절 3중 이상 | 한국어의 정상 좌분지 수식은 보존 | 두 문장이나 후치 설명으로 분할 |
| A-11 | 추상명사 + `~로 이어진다/이어지게 한다/연결된다` 반복 | 실제 순서·관계·지속을 뜻하면 보존 | 원문에 있는 구체 주체·결과를 직접 서술하거나 인과절로 풀기 |

`A-11`은 영어 `lead to`류를 만능 인과 동사로 옮긴 번역투와 AI식 결산에서 특히 주의한다. `성장으로 이어진다`, `변화를 이어지게 한다`처럼 추상 결과를 매번 같은 동사로 닫을 때 강한 신호로 올린다. 단순히 `연결된다`, `귀결된다`, `초래한다`로 바꾸지 말고 문장의 실제 인과를 드러낸다. 원문에 인과 근거가 없으면 새 인과를 만들지 않는다.

## B. 영어 병기·용어 과다

| ID | 신호 | 게이트 | 안전한 처방 |
| --- | --- | --- | --- |
| B-1 | 같은 영어를 괄호로 매번 병기 | 첫 정의·계약상 영문명이 필요한지 확인 | 첫 등장만 병기 |
| B-2 | 익숙한 한국어가 있는데 영어만 사용 | 업계 표준 약어·제품명은 보호 | 독자에게 자연스러운 표준어 사용 |
| B-3 | 영어 인용구로 문단을 장식 | 정확한 인용이면 보호 | 불필요하면 한국어 설명만 남김 |

## C. 기계적 구조·서식

| ID | 신호 | 게이트 | 안전한 처방 |
| --- | --- | --- | --- |
| C-1 | 모든 문단이 `첫째/둘째/셋째` 병렬 | 절차·요건 목록이면 보존 | 일부를 산문 흐름으로 연결 |
| C-2 | 짧은 불릿·헤딩의 과도한 반복 | 매뉴얼·요약 문서면 보존 | 의미 단위만 묶고 중복 헤딩 제거 |
| C-3 | 매 문단 첫 문장이 문단을 요약 | 보고서 관습인지 확인 | 장면·근거·직접 주장으로 시작 |
| C-4 | `먼저-반면-결국`, `A가 아니라 B` 공식 반복 | 논증에 꼭 필요한 대조는 보존 | 접속어를 덜고 문장 관계로 드러냄 |
| C-5 | `X: Y`, `X에서 Y로` 헤딩 공식 반복 | 실제 정보 구조인지 확인 | 짧은 평서형 제목으로 변경 |

## D. 상투어·결산·과장

| ID | 신호 | 안전한 처방 |
| --- | --- | --- |
| D-1 | `결론적으로/요약하면/정리하면` 반복 | 문단 자체가 결론이면 삭제 |
| D-2 | `시사하는 바가 크다/주목할 만하다/의미가 크다` | 구체 내용이 이미 있으면 삭제; 새 근거를 만들지 않음 |
| D-3 | `본질적으로/핵심적으로` 장식 | 뜻이 유지되면 삭제 |
| D-4 | `혁신적/획기적/압도적/강력한` 과장 반복 | 근거 없는 수식만 삭제; 수치나 사실을 새로 만들지 않음 |
| D-5 | `시대가 요구한다/기술이 묻는다` 같은 추상 주어 의인화 | 실제 주체가 원문에 있을 때만 복원 |
| D-6 | `지금이야말로 ~할 때다`, `~해야 할 것이다` 공식 결말 | 원래 주장 강도를 유지한 직접 종결 |

## E. 리듬·종결의 균일성

- **E-1** 비슷한 길이의 문장이 문단마다 같은 수로 반복되는지 본다.
- **E-2** 동일 종결어미가 네 문장 이상 기계적으로 이어지는지 본다.
- **E-3** 쉼표로 긴 절을 연속 연결하거나 연결어미 바로 뒤 쉼표를 반복하는지 본다.
- **E-4** 리듬을 바꿀 때 새 감탄, 반문, 단문을 일부러 만들지 않는다. 의미상 자연스럽게 합치거나 나눌 수 있는 문장만 조정한다.
- **E-5** 보고서·법률·학술 장르의 규칙적인 문장은 약한 신호로 낮춘다.

## F. 과잉 수식·추상명사

- **F-1** `매우/정말/대단히`가 근거 없이 반복되면 필요한 강조만 남긴다.
- **F-2** `중요하고 핵심적인` 같은 동의 수식은 하나로 줄인다.
- **F-3** `~적·~성·~화`가 연쇄된 명사 덩어리는 원문에 있는 동사·주체로 풀어 쓴다.
- **F-4** `기능과 역할`, `방향성과 목표`처럼 의미가 겹치는 짝을 확인한다. 서로 다른 뜻이면 유지한다.

## G. 과도한 완곡(hedging)

- **G-1** `~할 수 있을 것으로 보인다`, `~인 듯하다고 할 수 있다`처럼 완곡이 겹칠 때 한 단계만 남긴다.
- **G-2** 연구·법률·예측에서 불확실성은 정보다. 확실하지 않은 내용을 단언형으로 바꾸지 않는다.
- **G-3** `한편으로는/다른 한편으로는`, `장단점이 있다` 같은 균형 문구는 실제 양면 논증이면 보존한다.

## H. 접속사·메타 진입 반복

- **H-1** 문두의 `또한/따라서/즉/나아가/아울러/게다가`가 반복되면 문장 관계가 자명한 것부터 덜어낸다.
- **H-2** `이는/이 점에서/이 관점에서/이 말은`이 반복되면 가리키는 내용을 주어로 직접 연결한다.
- **H-3** 접속사를 다른 접속사로 기계적으로 치환하지 않는다. 삭제해도 관계가 명확한지 먼저 본다.

## I. 형식명사·의존명사

- **I-1** `~인 것이다/~한 것이다`, `~라는 점에 있다`, `~다는 의미다`가 반복될 때 직접 종결한다.
- **I-2** `것/점/바/수/데` 자체는 정상 한국어다. 밀도와 반복 문형을 본다.
- **I-3** `필요가 있다`, `가능성이 있다`, `능력을 갖추고 있다`는 뜻을 보존할 동사가 원문에 있을 때만 환원한다.

## J. 시각 장식

- **J-1** 볼드, 따옴표, 괄호, em dash, 이모지의 존재가 아니라 반복과 장르 부적합을 본다.
- **J-2** SNS·마케팅·메신저의 이모지와 줄바꿈은 장르 관습일 수 있다.
- **J-3** 인용 따옴표, 코드 백틱, Markdown 링크, 접근성을 위한 구조는 건드리지 않는다.

## K. 격상·강조 공식

AI 문체에서 자주 겹치는 강조 틀이다. 대조·범위·강조가 실제 정보면 보존하고, 같은 기능을 반복하는 틀만 직접 서술로 바꾼다.

| ID | 신호 | 게이트 | 안전한 처방 |
| --- | --- | --- | --- |
| K-1 | `단순한 A가 아니라 B` 반복 | A를 명시적으로 부정하는 논증인지 확인 | 불필요한 격상 틀을 빼고 B를 직접 서술 |
| K-2 | `중요한 것은 ~라는 점이다` | 초점 대조가 필요한지 확인 | 핵심 명제를 주어·서술어로 직접 표현 |
| K-3 | `~라는 점에서/측면에서 의미가 있다` | 평가 기준이 정보인지 확인 | 원문에 있는 성과와 평가를 직접 연결 |
| K-4 | `~뿐만 아니라 ~도` 연쇄 | 포함 범위가 논지인지 확인 | 자연스러운 나열이나 병렬로 단순화 |
| K-5 | 억지 삼분 구조와 `더 A, 더 B, 더 C` | 세 항목이 실제로 독립적인지 확인 | 중복 항목만 줄이고 필요한 항목은 유지 |

예: `중요한 것은 우리가 포기하지 않았다는 점입니다`는 별도 대조가 없다면 `우리는 끝내 포기하지 않았습니다`로 직접 쓸 수 있다. 그러나 `실수가 아니라 구조적 문제였다`처럼 분류 대조가 핵심이면 유지한다.

## L. 막연한 출처와 도구 흔적

| ID | 신호 | 안전한 처방 |
| --- | --- | --- |
| L-1 | `한 연구에 따르면`, `전문가들은` 뒤에 확인 가능한 출처가 없음 | 사실을 새로 단언하거나 삭제하지 말고 `확인 요망`으로 표시. 윤문만 요청받았다면 표현 유지 |
| L-2 | `[회사명]`, `[수치 입력]` 같은 미치환 자리표시자 | 추측해 채우지 말고 그대로 두며 확인 요청 |
| L-3 | `제가 아는 한`, `실시간 정보는 없지만` 같은 생성 도구 면책 | 글의 내용이 아니라 도구 잔재임이 명백할 때만 삭제 |
| L-4 | `utm_source=chatgpt.com` 같은 도구 추적 매개변수 | URL은 보호 요소다. 사용자가 링크 정리를 요청하지 않으면 변경하지 않음 |

원본에 있는 통계·인용을 허위라고 추정해 지우거나 경험담으로 바꾸지 않는다. 출처 검증은 별도 요청이 있을 때 수행한다.

## M. 과격식 중립 톤과 낮은 변동성

- **M-1** 비슷한 길이·구조·종결의 문장이 반복될 때 의미상 자연스럽게 합치거나 나눌 수 있는 범위에서만 리듬을 조정한다.
- **M-2** 원문에 있는 장면·수치·행위자가 추상 표현 뒤에 숨었으면 이를 앞으로 드러낸다.
- **M-3** 구체성이 부족하다는 이유로 새 장면, 수치, 경험, 구어체, 감정을 만들지 않는다.
- **M-4** 격식 장르의 중립 톤은 정상일 수 있다. 장르가 허용하고 원문에 근거가 있을 때만 더 직접적인 문장으로 바꾼다.
- **M-5** 담화표지 `-거든요·-잖아요·-더라고요·근데`를 사람다운 흔적으로 인위적으로 삽입하지 않는다.

## 편집 순서

`L 출처·도구 흔적 확인 → D 상투어 → K 격상 공식 → A 번역투 → I 형식명사 → H 접속사 → F 수식 → G 완곡 → B 영어 → C/J 구조·장식 → E/M 리듬·톤` 순서로 본다. 앞 단계에서 문장이 자연스러워졌으면 뒤 단계는 건너뛴다.

## 오탐 방지

- 격식, 유창성, 완결된 구조만으로 AI 글이라 판단하지 않는다.
- 한 표현의 등장 횟수를 목표치에 맞춰 억지로 줄이지 않는다.
- 광고, 공문, 법률, 학술, 매뉴얼의 정형성을 장르 관습과 분리한다.
- 사람 글에도 번역투와 상투어가 있다. 이 룰북은 출처 판별기가 아니다.
- 윤문 결과에 고의적 오류나 비정형성을 주입하지 않는다.
